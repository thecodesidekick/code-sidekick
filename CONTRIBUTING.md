# Contributing to Code Sidekick

First off, thank you for considering contributing to Code Sidekick! It's people like you that make Code Sidekick such a great tool for developers.

Even though the core engine of Code Sidekick is closed-source, your feedback, bug reports, and feature requests are the absolute lifeblood of our product roadmap.

## How Can I Contribute?

### Reporting Bugs
This section guides you through submitting a bug report for Code Sidekick. Following these guidelines helps maintainers and the community understand your report, reproduce the behavior, and find related reports.

- Use the **Bug Report** issue template.
- Ensure you have the latest version of the Code Sidekick extension installed.
- Provide clear steps to reproduce the issue.
- Include screenshots or screen recordings if they help explain the problem.

### Suggesting Enhancements
This section guides you through submitting an enhancement suggestion, including completely new features and minor improvements to existing functionality.

- Use the **Feature Request** issue template.
- Explain *why* this enhancement would be useful to most users.
- Provide a mockup or clear description of how you expect the feature to work.

### Participating in Discussions
The easiest way to contribute is simply by participating in our [GitHub Discussions board](https://github.com/thecodesidekick/code-sidekick/discussions).
- Upvote features you want to see built.
- Help other community members with questions they might have.
- Share workflows or prompts that work really well for you!

## Code of Conduct
This project and everyone participating in it is governed by the [Code Sidekick Code of Conduct](CODE_OF_CONDUCT.md). By participating, you are expected to uphold this code.
